# Visual Diff Cache Layer Fixes

## Overview

Fix critical bug and improve code quality in FCStd archive caching for visual feature diff. Keep cache root hard-coded at `/tmp/diffcad/visual_diff`. Skip cache eviction/cleanup logic.

## Architecture Decision (Adheres to `docs/Architecture.md`)

### Why not keep all logic inside action

`docs/Architecture.md` design principle: keep filesystem details at edges, use ports/adapters for external systems. Current action contains filesystem path construction, copy/extract logic, zip safety checks. That is infrastructure detail mixed into use-case orchestration.

### Proposed split

- **Application action (`open_visual_feature_diff.py`) owns:**
  - use-case orchestration
  - mapping old/new use-case inputs into generic FCStd revision inputs
  - failure mapping to `Result` and translation strings
- **Domain port + Infrastructure adapter owns:**
  - FCStd file location resolution
  - Archive materialization fallback
  - Extraction and zip-slip validation
  - Directory creation/refresh behavior
  - Internal storage layout (hidden from application)

This keeps action small/stateless, pushes filesystem operations to infra edge, and respects domain-layer port placement per architecture rules.

### Minimal complexity approach

Do not add broad new subsystem. Add one focused port in domain layer + one focused adapter in infrastructure for visual-diff FCStd source preparation only.

- Port location: `freecad/diff_wb/domain/freecad_ports.py` (adds to existing FreeCAD-facing ports)
- Adapter location: `freecad/diff_wb/infrastructure/freecad/visual_diff_fcstd_source.py`
- Wire in DI container with existing action dependencies

**Important:** No "cache" terminology appears in application or domain layers. The port speaks in terms of "FCStd revision preparation". Cache is purely an internal implementation detail of the adapter.

**Layer rule:** Do not put application request types in domain. Domain port methods must accept domain-layer models (`GitRepository`) and primitives only. Application action adapts `OpenVisualFeatureDiffRequest` into those arguments.

No cleanup/eviction policy added.

---

## Problems Identified

### Critical Bug

When `should_reuse=True`, code can skip materialization without validating archive file presence. If archive was deleted but directory remains, extraction path handling becomes invalid and behavior degrades.

### Path Structure Inconsistency

Commit cache uses `{hash}/...`; index/working use `staging|working/...`. Mixed semantics harder to reason about.

### Missing Archive Verification

No explicit archive existence gate before reuse extraction path.

### Test Smell

Unit test accesses `fake_port._file_bytes.clear()` directly (private state).

## Proposed Changes

### 1) Introduce FCStd revision preparation port + adapter

#### New port protocol in domain layer

Add to `freecad/diff_wb/domain/freecad_ports.py`:

```python
class FreeCadFcstdDiffSourcePort(Protocol):
    def prepare_fcstd_revision(
        self,
        repo: GitRepository,
        git_path: str,
        commit: str | None,
        working_tree_document_path: str | None = None,
    ) -> Path | None:
        """Materialize and extract requested revision; return extraction root path.

        Returns None if materialization or extraction fails.
        commit=None and working_tree_document_path=None means index revision.
        working_tree_document_path takes precedence for working tree revision.
        Internal implementation handles source selection (index/working/commit),
        storage layout, archive reuse, and safe extraction.
        """

    def find_brep(self, extract_root: Path, brep_name: str) -> Path | None:
        """Find BREP file by name inside extracted FCStd tree.

        Returns path if found, None otherwise.
        """
```

Reasoning:
- No application-layer request object crosses into domain.
- No `Literal["old", "new"]` leaks use-case language into domain.
- Same method supports old and new revisions through generic inputs.
- Adapter can own source resolution and internal storage behavior.

#### New infrastructure adapter

`freecad/diff_wb/infrastructure/freecad/visual_diff_fcstd_source.py`

Responsibilities:
- Resolve revision source (index, working tree document, or commit)
- Compute internal storage paths under hard-coded root
- Materialize FCStd from git ref/index or copy from working tree
- Reuse existing archives for commit revisions when present
- Always refresh ephemeral revisions (index/working)
- Extract FCStd safely with zip-slip validation
- Find BREPs inside extracted tree

Top comment required by repo rule:
- `# File responsibility: FCStd revision preparation for visual diff (storage, extraction, BREP lookup).`

### 2) Internal storage directory structure change

**Note:** This structure is internal to the adapter. Application layer does not know about it.

Current:

```text
/tmp/diffcad/visual_diff/
├── staging/doc.FCStd
├── working/doc.FCStd
└── abc1234/doc.FCStd
```

Target:

```text
/tmp/diffcad/visual_diff/
├── index/doc.FCStd
├── working/doc.FCStd
└── commits/abc1234/doc.FCStd
```

Rules:
- `index` and `working` are ephemeral: always rematerialize + reextract
- `commits/<short_hash>` is persistent: reuse archive if present, materialize if missing

### 3) Action refactor (orchestration-only)

Refactor `OpenVisualFeatureDiffAction`:
- Remove commit ref resolution from action; adapter resolves refs internally through `GitService`.
- Replace all internal path/extraction/BREP-finding logic with port calls:
  - `prepare_fcstd_revision(repo, git_path, old_commit, None)` -> extract_root_old
  - `prepare_fcstd_revision(repo, git_path, new_commit, working_tree_document_path)` -> extract_root_new
  - `find_brep(extract_root_old, brep_name)`
  - `find_brep(extract_root_new, brep_name)`
- Remove methods: `_old_side`, `_new_side`, `_commit_side`, `_side_workspace`, `_extract_fcstd_safely`, `_find_file_by_name`
- Remove internal dataclass `_VisualDiffSide`
- Remove materialization callbacks from action; adapter materializes from git/index or working tree path.
- Maintain existing `Result` failure mapping and logging behavior

### 4) Critical bug fix behavior

For commit revisions:
- If archive exists at internal storage path, reuse without git read
- If archive missing, materialize from git ref as fallback
- Extraction happens only after archive presence confirmed
- If materialization fails for either revision, return `VISUAL_DIFF_MISSING_FCSTD_MESSAGE`

This logic lives entirely in the infrastructure adapter. Application receives `Path | None`.
